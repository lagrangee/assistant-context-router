# Doc Layering

## 目标
定义项目文档的最小分层，避免协作过程、稳定结论、执行说明、历史资料混在一起。

## 分层
### docs map
- 入口索引
- 回答先读什么、哪里是权威

### plan
- 稳定结论
- 策略边界
- acceptance
- decision

### progress
- 当前协作状态
- 阶段进展
- 收尾与验证

### archive
- 不再作为当前真相的历史材料

## 新增规则
新增文档前先判断：
- 如果是稳定结论，进入 `plan/`
- 如果是协作过程，进入 `progress/`
- 如果是历史材料，进入 `archive/`

## 升格规则
当 progress 中某条结论已经足够稳定，可作为后续实现或评审引用时，应升格到 `plan/`。

## 归档规则
当某份文档不再表达当前真相，但仍需保留时，移入 `archive/`。
