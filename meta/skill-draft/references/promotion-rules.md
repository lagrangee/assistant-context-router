# Promotion Rules

## 目标
定义哪些内容可以从项目内候选规范升格为正式文档，哪些内容可以进一步升格为跨项目通用 skill。

## 从协作面板升格到正式文档
当一条内容满足以下条件时，建议从 `progress/collab.md` 升格到正式文档：
- 已形成明确 scope
- 已有 acceptance 或 decision rule
- 已不再只是临时沟通内容
- 后续实现或评审会反复引用它

## 从项目内候选规范升格到通用 skill
只有在以下条件满足后，才建议升格：
- 已在两个以上项目中验证
- 已在两类以上外部 agent 协作中验证
- 已覆盖不止一种任务类型
- 文档分层与 request/reply 规则已证明稳定
- skill 边界已足够清楚，不会吞掉项目自己的文档职责

## 不应升格的内容
以下内容不应进入通用 skill：
- 某个项目的业务背景
- 某个项目的专属 step 结论
- 某个项目的 docs map 实体内容
- 某个 agent 的单独偏好实现

## 原则
- 先在项目内跑顺，再抽象
- 先多样本验证，再打包 skill
- 通用 skill 只沉淀方法，不复制项目事实
